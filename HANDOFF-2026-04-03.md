## 7. Critical Implementation Patterns

### API routes must NOT use `export const runtime = 'edge'`
The `open-next.config.ts` uses `wrapper: "cloudflare-node"`. Routes tagged `edge` compile to pure edge runtime which breaks `@supabase/supabase-js`. Never add this annotation to API routes.

### `supabase.auth.admin` cannot be called client-side
Requires service role key â€” server-only. Always use a dedicated `/api/admin/` route.

### Zustand store vs Supabase
For mutations: (1) call the lib function to persist to Supabase, (2) refresh data from Supabase, (3) optionally update Zustand for optimistic UI. The UI tables are driven by Supabase data, not Zustand arrays.

### Farmer dropdown includes admins
Both `CreateFarmModal` and `ClaimReportModal` receive `[...farmers, ...admins]` as owner list.
Name rendering: `farmer.full_name || farmer.name || farmer.email`

### React-Leaflet map centre is NOT reactive
Use `FlyToLocation` inner component in `MapInner.tsx` for programmatic map movement.

### Geo-filtering for farmer reports
`isReportNearFarm(report, farm)` in `appStore.ts`: uses ray-casting point-in-polygon for reports inside field boundaries + `distanceToSegmentMeters` for buffer distance check. Farmers only see reports when they have at least one farm with mapped fields.

### Farms are Supabase-backed
All farm CRUD goes through `src/lib/supabase-client.ts`. Zustand farm state is loaded from Supabase on dashboard mount via `loadFarms()`. No more localStorage-only farm state.

### Password reset flow
Three branches in `reset-password/page.tsx`:
1. `token_hash` + `type=recovery` â†’ `verifyOtp()` (PKCE, current Supabase default)
2. `code` â†’ `exchangeCodeForSession()`
3. Legacy hash `access_token` â†’ `getSession()` with timeout

Supabase email template body must use:
```
{{ .SiteURL }}/auth/reset-password?token_hash={{ .TokenHash }}&type=recovery
```

### Audit logging
`writeAuditLog()` in `audit.ts` is fire-and-forget. Never `await` it in a way that blocks the user action. The `audit_logs` table has no UPDATE/DELETE RLS â€” immutable append-only.

### `condition` field is free text
Migration 006 removed the CHECK constraint on `sheep_reports.condition`. Do not add a new one.

### npm / node PATH
```bash
export PATH="/usr/local/bin:$PATH"
```
