### Push notifications ÔÇö during auth setup

- After successful sign-in **and** sign-up, `Notification.requestPermission()` is called before the router redirect
- This works because the form-submit handler is a user gesture ÔÇö Android shows the native OS permission dialog at exactly this point
- If the user grants, `PushPermissionBanner` silently calls `subscribeToPush(userId)` on first dashboard mount (it already checks `permission === 'granted'` on mount)
- If the user denies or is on a browser that doesn't support it, the try/catch swallows the error and the flow continues normally

### "Keep me signed in" checkbox

- Added to **both** sign-in and sign-up forms
- Defaults to `true` (Supabase auto-persists sessions anyway)
- Label: "Keep me signed in on this device ÔÇö required for offline report sync"
- If unchecked, writes `lbp-session-temp` to localStorage as a flag (can be used in future to warn the user at sync time)
- Session persistence behaviour is unchanged technically (Supabase localStorage storage)

### Offline + logged-out access from landing page

This fixes the critical scenario: PWA installed, user is logged out, device loses connectivity.

**Before:** logged-out users saw the landing page with no ability to report offline at all.

**After:**

| State | What user sees |
|---|---|
| Offline + logged out | Plum banner at top: "You're offline ÔÇö you can still save a report for later" + "Save report" button Ôćĺ opens `OfflineCapture` overlay |
| Online + logged out + pending reports | Amber banner: "You have N saved reports ÔÇö sign in to upload" + direct Sign In link |
| Online + logged in | No banner (normal WalkerDashboard renders instead of LandingPage) |

**Implementation:**
- `LandingPage` in `src/app/page.tsx` now imports `useOnlineStatus`, `OfflineCapture`, `getPendingCount`
- `pendingCount` loaded from IndexedDB on mount and after each capture
- `OfflineCapture` renders as a full-screen modal overlay above the landing page when `showOfflineCapture` is true
- The existing "not signed in" warning inside `OfflineCapture` still tells the user to log in before syncing
- `OfflineSyncBanner` inside `WalkerDashboard` handles the actual sync automatically once they log in

**Known constraint:** If the device has never loaded the app while online, `reportCategories` will be empty and the category picker shows only the hardcoded Sheep option. This is unavoidable without a local category cache.
