## 1. What This Project Is

Little Bo Peep connects countryside walkers who spot stray livestock with the farmers who need to recover them. Walkers submit geo-tagged reports with photos. Farmers claim and resolve reports. Admins manage the platform, users, categories, and billing.

## 7. Report & Category System — Full Current State

### 7.1 Report data model (SheepReport)

Key fields on every report:
- `categoryId` / `categoryName` / `categoryEmoji` — stored at submission time
- `categoryImageUrl` — stored at submission time; used for map markers (new in migration 015)
- `conditions: string[]` — multi-select array (new); `condition: string` kept for backward compat
- `mapSnapshotUrl` — OSM viewer URL captured at step 1 completion
- `status` — `'reported' | 'claimed' | 'resolved' | 'escalated' | 'complete'`
- `screeningRequired` — boolean; admin can flag for screening queue
- Extended attribution: `submittedByUserName`, `roleOfSubmitter`, `affectedFarmIds[]`, `affectedFarmerIds[]`

### 7.2 Category data model (ReportCategory)

```typescript
{
  id, name, emoji, description?, conditions[], showCount, countLabel,
  isActive, sortOrder, subscriptionMode, imageUrl?, nameTranslations?, createdAt
}
```

- `imageUrl` — Supabase Storage URL for category icon; overrides emoji on map markers and category pickers
- `nameTranslations` — `Record<string, string>` keyed by language code (e.g. `{ cy: 'Defaid', ga: 'Caoire' }`). English (`name`) is always the fallback. Use `getLocalizedCategoryName(category, lang)` from appStore to display.
- `subscriptionMode` — `'compulsory' | 'default_on' | 'default_off'` — controls farmer notification defaults

### 7.3 Walker report flow

**Step 1 — Location**
- Map shows recent reports with correct category icon (image or emoji)
- "Selected" drop-pin uses category image or emoji
- Nearby report warning within 100m
- `mapSnapshotUrl` stored at step 1 completion

**Step 2 — Details**
- Quantity: quick-pick grid (1–10) + stepper (−/input/+)
- Condition: multi-select chips (toggleable; all selected by default)
- Category-specific conditions from `activeCategory.conditions`

**Step 3 — Contact**
- Pre-filled for logged-in users

**Step 4 — Confirm + Submit**

### 7.4 Category picker (Walker)
- Opens as bottom sheet (mobile) / centred modal (desktop)
- Each category shows its image (if set) or emoji, name, and description
- ★ star button sets a category as default (persisted to `localStorage` key `lbp-preferred-category`)
- FAB icon reflects the preferred category's image or emoji

### 7.5 Admin — Reports tab

- Clickable category icon (image or emoji) opens report detail
- Reporter name and role shown on each row
- Consistent button heights (`px-3 py-1.5 rounded-lg text-xs font-medium`)
- Export: CSV, Excel (SheetJS), PDF (jsPDF) — loaded from CDN at export time
- Map snapshot URL link in detail view
- **Edit** — admin can edit any report's fields and photos
- **Comments** — admin can add manual comments; system comments logged automatically on status changes

### 7.6 Admin — Categories tab

- Create / edit / delete categories
- **Image uploader**: upload a custom icon to Supabase Storage (preferred over emoji)
- **Emoji fallback**: large picker grid if no image
- **Name translations**: per-language name inputs for all enabled non-English languages (English is required and used as fallback)
- **Condition chips**: add/remove condition options
- **Subscription mode**: Compulsory / Default On / Optional
- **Reorder modal**: drag-and-drop to reorder all categories

### 7.7 Map markers

`getMarkerIcon(type, status, emoji, imageUrl)` in MapInner.tsx:
- `imageUrl` takes priority over `emoji` for both 'sheep'/'existing' and 'selected' types
- Border colour reflects status: green (reported), yellow (claimed), grey (resolved)
- Both `Map.tsx` and `MapInner.tsx` have `imageUrl` in the marker props interface

### 7.8 Report comments (report_comments table)
- `comment_type: 'manual' | 'system'`
- Manual: admin writes a comment in the detail panel
- System: auto-logged by `writeSystemComment()` on status changes (fire-and-forget)
- Functions: `fetchReportComments(reportId)`, `addReportComment(reportId, userId, text, type)`

## 8. Critical Implementation Patterns

### API routes must NOT use `export const runtime = 'edge'`
`open-next.config.ts` uses `wrapper: "cloudflare-node"`. Edge-tagged routes break `@supabase/supabase-js`.

### `supabase.auth.admin` is server-only
Requires service role key. Always route through `/api/admin/` server routes.

### React-Leaflet map centre is NOT reactive
Use `FlyToLocation` inner component in `MapInner.tsx` which calls `map.flyTo()` imperatively.

### Zustand + Supabase mutation pattern
1. Call the `supabase-client.ts` function to persist
2. Reload from Supabase (or optimistic update)
3. Zustand is the UI's reading layer, not the write authority

### Worktree / deploy path
```bash
# Working directory for this session:
/Users/chrissavides/Documents/littlebopeep-repo/.claude/worktrees/wonderful-shirley
# Deploy from this path:
npm run deploy
```

### Commit before deploy (always)
Commit all changes to git before running `npm run deploy`. The build artifacts in `.open-next/` will drift from source otherwise.

### `condition` field is free text
No CHECK constraint (removed in migration 006). Never add one back. `conditions[]` array stores multi-select values; `condition` (singular) is kept for backward compat.

### Audit logging
`writeAuditLog()` in `audit.ts` is fire-and-forget — never `await` it in a way that blocks user action. The `audit_logs` table is append-only (no UPDATE/DELETE RLS).

### Farmer dropdown includes admins
`CreateFarmModal` and `ClaimReportModal` receive `[...farmers, ...admins]`. Name rendering: `farmer.full_name || farmer.name || farmer.email`.

### Category name localisation
Use `getLocalizedCategoryName(category, currentLanguage)` (exported from `appStore.ts`) anywhere you display a category name. Returns translated name or falls back to English `category.name`.

## 10. Known Issues / Not Yet Done

1. **Category `nameTranslations` not yet used for display** — `getLocalizedCategoryName()` is exported and ready. It needs to be wired into:
   - WalkerDashboard category picker (category names shown to walkers)
   - FarmerDashboard subscription toggles
   - Report confirmation step (category name in summary)
   - Admin display of category names in reports list

2. **`subscription_mode` column may not exist in Supabase** — The app reads `(c as any).subscription_mode || 'default_on'` as a fallback. If the column doesn't exist yet, add it:
   ```sql
   ALTER TABLE report_categories ADD COLUMN IF NOT EXISTS subscription_mode TEXT DEFAULT 'default_off';
   ```

3. **Walker edit flow** — `editOwnReport` in store calls `updateReport` in supabase-client. Works for text fields. Photo re-upload within edit is not yet implemented.

4. **PWA icons missing** — `public/manifest.json` references `/icons/icon-192.png` and `/icons/icon-512.png` which don't exist. App installs but shows a blank icon.

5. **Photo sync in offline mode** — Photos stored as base64 in IndexedDB. Sync uploads only metadata. Future: convert data URLs to Blob and upload to Supabase Storage.

6. **Audit log nav tab** — Removed from admin nav to prevent scroll (too many tabs). View and DB writes are intact. Re-add when nav layout supports more tabs.

7. **Translations for new strings** — Strings added in April 2026 sessions (category subscription modes, notification preferences, status labels, report comments UI) need Welsh (cy), Irish (ga), and Scottish Gaelic (gd) translations added via the Supabase `translations` table.

8. **`conditions[]` not yet stored in offline captures** — `OfflineCapture.tsx` uses the old `condition` (single string). Future: update to use multi-select conditions.
