## 4. Deploy process (complete reference)

```bash
# 1. Edit source files in src/
# 2. Commit all changes BEFORE deploying
git add src/path/to/changed-file.tsx
git commit -m "Brief description"

# 3. Deploy — this runs: opennextjs-cloudflare build && wrangler deploy
npm run deploy
```

`npm run deploy` is defined in `package.json` as:
```json
"deploy": "opennextjs-cloudflare build && wrangler deploy"
```

`open-next.config.ts` uses `wrapper: "cloudflare-node"`.  
**Never** add `export const runtime = 'edge'` to API routes — it breaks `@supabase/supabase-js`.

## 7. Data models

### SheepReport — notable fields
```typescript
categoryId, categoryName, categoryEmoji    // captured at submission time
categoryImageUrl                           // category image URL for map markers (migration 015)
conditions: string[]                       // multi-select; condition: string kept for backward compat
mapSnapshotUrl                             // OSM viewer URL captured at step 1
status: 'reported'|'claimed'|'resolved'|'escalated'|'complete'
screeningRequired: boolean
```

### ReportCategory — notable fields
```typescript
imageUrl?                                  // Supabase Storage URL — overrides emoji everywhere
nameTranslations?: Record<string,string>   // e.g. { cy: 'Defaid', ga: 'Caoire' }
descriptionTranslations?: Record<string,string>
conditionTranslations?: Record<string, Record<string,string>>  // { cy: { 'Healthy': 'Iach' } }
subscriptionMode: 'compulsory'|'default_on'|'default_off'
```

Use `getLocalizedCategoryName(category, langCode)` from `appStore.ts` to display names with fallback to English.

### report_comments
```typescript
comment_type: 'manual'   // admin wrote it in the detail panel
comment_type: 'system'   // auto-logged by writeSystemComment() on status changes
```
Functions in `supabase-client.ts`: `fetchReportComments(reportId)`, `addReportComment(reportId, userId, text, type)`

## 8. Admin dashboard — reports tab features

- **Default sort**: Latest first (was "longest unclaimed")
- **Pagination**: Per-page selector (25 / 50 / 100 / All) — default 25. First / Prev / Page X of Y / Next / Last controls. Page resets to 1 on any filter change.
- **Select all**: Selects current page only
- **Filters**: Status, archive state, farmer, farm, date range, keyword
- **Export**: CSV / Excel (SheetJS via CDN) / PDF (jsPDF via CDN)
- **Category icon**: Clickable — shows category image or emoji, opens detail panel
- **Reporter info**: Shown on each row
- **Detail panel**: Edit fields + photos, manual comments, system comment log, map snapshot link
- **Duplicate warning**: Only triggers for same-category reports within 100m

## 11. Critical implementation rules

| Rule | Why |
|---|---|
| **Commit before deploy** | `.open-next/` is gitignored; deploying uncommitted work leaves git history wrong |
| **No `runtime = 'edge'`** | Breaks `@supabase/supabase-js` under Cloudflare edge runtime |
| **`supabase.auth.admin` is server-only** | Needs service role key — always use `/api/admin/` routes |
| **Map centre is not reactive** | Use `FlyToLocation` inner component in `MapInner.tsx` |
| **Zustand is the read layer** | Mutations go: `supabase-client.ts` → reload from Supabase → optional Zustand update |
| **`condition` field is free text** | No CHECK constraint (dropped migration 006). `conditions[]` is the multi-select array. |
| **Audit log is append-only** | `writeAuditLog()` is fire-and-forget. No UPDATE/DELETE on `audit_logs`. |
| **Farmer dropdowns include admins** | Pass `[...farmers, ...admins]` as owner list in farm/claim modals |
